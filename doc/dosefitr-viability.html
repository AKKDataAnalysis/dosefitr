Overview

This vignette walks through the viability / proliferation dose-response pipeline of dosefitr, end-to-end, on bundled example data. The pipeline is single-channel (CellTiter-Glo-style luminescence), in contrast to the dual-channel NanoBRET pipeline covered in vignette("dosefitr-nanobret"). The plate layout is the same – this matters because the two pipelines compose: one set of compounds, one set of plate maps, two readouts.

The seven stages of the pipeline:

  1. Setup and stage the bundled plates into a working directory.
  2. Inspect the plate layout and the info table.
  3. Process the raw counts into normalised viability tables (batch_viability_analysis()).
  4. Fit 4PL dose-response curves (batch_drc_analysis() with normalize = TRUE).
  5. Visualise individual and overlay plots (batch_save_all_drc_plots(), plot_multiple_compounds(), compare_plates_drc()).
  6. Export Scarab-format tables (scarab_viability()).
  7. Read the quality metrics.

A short note at the end (Section 8) explains how to use the v2 processor when your info table follows the alternative PHERAstar-with-merged-controls format.

1. Setup

library(dosefitr)

The package ships with two viability plates and a single info table covering them both. They are located in inst/extdata/ and are loaded with system.file():

extdata_dir <- system.file("extdata", package = "dosefitr")

list.files(extdata_dir, pattern = "^viability_")
#> [1] "viability_info.xlsx"     "viability_plate_01.xlsx"
#> [3] "viability_plate_02.xlsx"

batch_viability_analysis() scans a directory for plate files matching a regular expression, so we stage the relevant files into a working directory under tempdir(). This keeps the vignette CRAN- compliant (no writes outside tempdir()):

work_dir <- file.path(tempdir(), "dosefitr_viability")
dir.create(work_dir, showWarnings = FALSE)

invisible(file.copy(
  from      = list.files(extdata_dir, pattern = "^viability_",
                         full.names = TRUE),
  to        = work_dir,
  overwrite = TRUE
))

list.files(work_dir)
#> [1] "viability_info.xlsx"     "viability_plate_01.xlsx"
#> [3] "viability_plate_02.xlsx"

2. Plate layout and info table

Each plate is a 384-well PHERAstar luminescence export with a single emission channel. The first 12 rows of the worksheet are instrument metadata; the data block follows at row 13. Each of the 16 rows (A-P) holds one compound:

  • Column 1 – DMSO (vehicle) control, the “100 % viability” baseline.
  • Columns 2-12 – 11 inhibitor concentrations, technical replicate 1.
  • Columns 13-23 – the same 11 concentrations, technical replicate 2.
  • Column 24 – compound-saturated wells, the “0 % viability” floor.

viability_info.xlsx carries one sheet per plate. The first column holds log10-concentrations (with NA for the control rows), the remaining columns identify each plate row:

info_path <- file.path(work_dir, "viability_info.xlsx")

info_plate_01 <- openxlsx::read.xlsx(info_path, sheet = "plate_01")

3. Viability table processing

batch_viability_analysis() reads both plates, normalises each row to its DMSO control, applies the 0 % floor at the saturated-compound control, and produces percent-viability tables ready for fitting. For the bundled v1-format data we use version = "v1":

via_out <- file.path(tempdir(), "dosefitr_viability_proc")
dir.create(via_out, showWarnings = FALSE)

via_res <- batch_viability_analysis(
  directory        = work_dir,
  info_file        = "viability_info.xlsx",
  data_pattern     = "viability_plate_\\d+\\.xlsx$",
  control_0perc    = 1,
  control_100perc  = 24,
  selected_columns = 1:24,
  generate_reports = FALSE,
  output_dir       = via_out,
  verbose          = FALSE,
  version          = "v1"
)

names(via_res)
#> [1] "plate_01" "plate_02"

Note that viability accepts integer control column indices, unlike NanoBRET which uses character indices.

The returned object mirrors the NanoBRET pipeline: one entry per plate, with the processed table under $result$modified_ratio_table:

4. Dose-response curve fitting

batch_drc_analysis() fits each compound with a 4PL model. For viability data the standard call passes normalize = TRUE, which rescales the raw counts to percent and clamps the fits to viability- plausible Bottom and Top windows.

drc_out <- file.path(tempdir(), "dosefitr_viability_drc")
dir.create(drc_out, showWarnings = FALSE)

drc_res <- batch_drc_analysis(
  batch_results    = via_res,
  model            = "4pl",
  normalize        = TRUE,
  output_dir       = drc_out,
  generate_reports = FALSE,
  verbose          = FALSE
)

drc_res$drc_results$plate_01$drc_result$successful_fits
#> [1] 16

5. Visualisation

5.1 Save one plot per compound (and a combined panel per plate)

batch_save_all_drc_plots() is the workhorse plotting function for a batch result. It calls plot_dose_response() on every successful fit, writes one PNG per compound, and assembles the per-plate compounds into a single panel image with patchwork:

plots_dir <- file.path(tempdir(), "dosefitr_viability_plots")
dir.create(plots_dir, showWarnings = FALSE)

results2 <- batch_save_all_drc_plots(
  batch_drc_results = drc_res,
  output_dir        = plots_dir,
  verbose           = FALSE,
  dpi               = 120,            # lighter for the vignette
  panel_width_per_col  = 4,
  panel_height_per_row = 4,
  panel_spacing        = 1.2
)

results2$successes
#> [1] 32
basename(results2$panel_files)
#> [1] "plate_01_panel.png" "plate_02_panel.png"

plot_multiple_compounds() is the canonical way to compare several compounds in one figure, ranked by potency:

plot_multiple_compounds(

  compound_indices = 1:4,
  plate            = "plate_01",
  show_error_bars  = TRUE,

5.3 The same compound across plates

compare_plates_drc() makes one figure per construct-compound pair that appears on more than one plate – the inter-plate consistency check. The bundled plate_02 repeats four of plate_01’s compounds under the same construct, so four cross-plate plots are produced:

compare_out <- file.path(tempdir(), "dosefitr_viability_compare")
dir.create(compare_out, showWarnings = FALSE)

cmp_res <- compare_plates_drc(
  batch_drc_result = drc_res,
  compare_by       = "compound",
  output_dir       = compare_out,
  min_plates       = 2,
  verbose          = FALSE
)

list.files(compare_out, pattern = "\\.png$")
#> [1] "KinaseA__Cpd1.png"  "KinaseA__Cpd5.png"  "KinaseB__Cpd13.png"
#> [4] "KinaseB__Cpd9.png"

The returned object’s $plots list has one ggplot per pairing.

7. Reading the quality metrics

The output of batch_viability_analysis() includes per-plate quality metrics, printed at run time when verbose = TRUE. Key numbers:

  • S/B – ratio of mean 100 % control wells (DMSO) to mean 0 % control wells (compound-saturated). Above 5 is usable.

  • Z-prime – robustness statistic combining S/B with control variability. Above 0.5 is usable, above 0.7 is excellent.

  • CV % – coefficient of variation between technical replicates. Under 10 % is well-behaved.

The fit-level Curve_Quality column carries one of:

  • Good curve – everything passed.
  • Wide logIC50 CI range – the IC50 95 % CI is wider than the test concentration range; the IC50 is at or beyond a tested concentration.
  • Top too low / Bottom too high – the fitted plateaux do not bracket the response range.

See ?batch_drc_analysis for the complete list and thresholds.

8. Note on the v2 processor

For info tables that follow the alternative PHERAstar layout (with merged control columns in a single sheet), pass version = "v2":

via_res_v2 <- batch_viability_analysis(
  directory        = work_dir,
  info_file        = "viability_info.xlsx",
  data_pattern     = "viability_plate_\\d+\\.xlsx$",
  control_0perc    = 1,
  control_100perc  = 24,
  selected_columns = 1:24,
  generate_reports = FALSE,
  output_dir       = via_out,
  version          = "v2"
)

The two processors share the downstream pipeline: the same batch_drc_analysis() call, the same plot functions, the same scarab_viability() export. The only structural difference is the shape of the info table.
